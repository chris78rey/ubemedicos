#### Impacto y riesgos

Se deja **hecho el siguiente paso de backend** de la secuencia: publicar y probar el bloque de **pagos demo**.

El repositorio empacado que se recibió contiene solo backend; no trae carpetas reales de frontend/Nuxt dentro del repomix. Por eso, la opción conservadora y correcta es cerrar primero este backend y no inventar pantallas web fuera del código entregado.

Riesgos controlados:

* no hay migraciones nuevas;
* no se altera el esquema actual;
* no se toca el CRUD de disponibilidad;
* no se cambia autenticación;
* se evita duplicidad del pago pendiente;
* se deja reversión simple por archivos.

#### Prueba de Fase 1

Se espera que:

* el paciente solo pueda generar pago para su propia cita;
* no se creen dos pagos pendientes para la misma cita;
* un pago exitoso marque `is_paid=True`;
* la cita pase a `confirmed` cuando proceda;
* un pago fallido deje la cita sin pago.
