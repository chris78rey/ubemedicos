#### Impacto y riesgos

Se dejó **hecho el paquete ejecutable** para aplicar el bloque de pagos demo sobre el proyecto local.

No se agregó migración nueva.
El cambio solo toca:

* `backend/apps/payments/api_views.py`
* `backend/apps/payments/api_urls.py`
* `backend/apps/payments/tests_api.py`
* `backend/config/urls.py`

Riesgo controlado:

* no cambia el esquema;
* no toca disponibilidad;
* no toca autenticación;
* deja reversión por backup automático.

#### Fase 1

**Objetivo**

Aplicar el bloque `payments demo` directamente en `G:\codex_projects\ubemedicos`.

**Cómo aplicar**

En PowerShell, dentro de la carpeta descomprimida:

```powershell
Set-ExecutionPolicy -Scope Process Bypass
.\APLICAR_PAYMENTS_DEMO.ps1
```

Si el repo está en otra ruta:

```powershell
Set-ExecutionPolicy -Scope Process Bypass
.\APLICAR_PAYMENTS_DEMO.ps1 -RepoRoot 'D:\otra_ruta\ubemedicos'
```

**Qué hace**

* crea backup automático;
* copia los archivos nuevos;
* reemplaza `backend/config/urls.py`;
* deja el bloque listo para probar.

#### Prueba de Fase 1

```powershell
cd G:\codex_projects\ubemedicos\backend
python manage.py check
python manage.py test apps.payments.tests_api -v 2
```

#### Plan de reversión total

El script de aplicación genera una carpeta tipo:

```powershell
backup_before_apply_YYYYMMDD_HHMMSS
```

Para revertir:

```powershell
Set-ExecutionPolicy -Scope Process Bypass
.\REVERTIR_PAYMENTS_DEMO.ps1 -BackupDir '.\backup_before_apply_YYYYMMDD_HHMMSS'
```

Después:

```powershell
cd G:\codex_projects\ubemedicos\backend
python manage.py check
```

El siguiente paso, una vez aplicado esto, sería montar el frontend real de pagos o el panel admin de pagos.
